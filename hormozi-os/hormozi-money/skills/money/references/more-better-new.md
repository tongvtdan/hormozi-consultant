# More / Better / New — Reference
**Source: ACQ Scaling + Money Models Framework — Alex Hormozi**

> "More before Better before New. Every time. No exceptions."
> "Adding 'new' before you've maxed 'more' is the fastest way to destroy a working business."

---

## The Framework Explained

### Phase 1 — MORE

**Definition:** Do more of the exact same thing that is already working.

**When you are here:**
- The current approach generates revenue but not at full capacity
- You haven't exhausted the available market with the current strategy
- Volume is inconsistent week-to-week
- You're changing tactics before the current one is maximized

**What "more" looks like:**
- Running warm outreach at 5 touches/day → increase to 20
- Posting content 2×/week → increase to daily
- Taking 5 sales calls/week → take 10
- Running ads at $1,000/month → test $3,000/month (if LTV:CAC is ≥ 3:1)

**What "more" does NOT mean:**
- Hiring more before your process is documented
- Adding more channels before the primary one is maxed
- Creating more offers before the core offer is proven

**More graduation criteria:**
- [ ] Primary channel running at full capacity
- [ ] Revenue is consistent (< 20% variance week-to-week)
- [ ] ≥ 10 paying customers from the primary approach
- [ ] Execution is identical every week (playbook exists and is followed)

---

### Phase 2 — BETTER

**Definition:** Improve the quality, efficiency, or conversion of the existing approach.

**When you are here:**
- Volume is consistent — but results could be better
- Close rate is below 30%
- Customer results have variance
- LTV:CAC ratio is below 3:1
- Churn is above industry standard
- Delivery takes more effort than it should

**What "better" looks like:**
- Improving the sales script (higher close rate)
- Improving the lead magnet (higher opt-in rate)
- Improving client onboarding (lower churn)
- Raising prices with a stronger value stack (higher revenue per customer)
- Systemizing delivery (same results, less founder time)

**What "better" does NOT mean:**
- Adding a second product before the first is optimized
- Changing the audience before the offer is proven
- Rebuilding the funnel when copy is the actual problem

**Better graduation criteria:**
- [ ] Close rate on qualified calls ≥ 30%
- [ ] LTV:CAC ratio ≥ 3:1
- [ ] NPS ≥ 50
- [ ] Churn within industry norms
- [ ] Delivery runs without founder in the critical path

---

### Phase 3 — NEW

**Definition:** Add a new channel, offer, product, or revenue stream.

**When you are here:**
- Current channel is running at maximum volume
- Quality is optimized (Better criteria all met)
- TAM in current niche is feeling constrained
- Cash and team resources can support a new initiative without harming the core

**What "new" looks like:**
- Adding paid ads after warm + organic are maxed
- Adding cold outreach after warm is proven
- Launching a continuity offer after the core offer is optimized
- Adding a second ICP or niche (adjacent expansion)
- Launching a new product tier (premium or entry)
- Entering a second distribution channel (LinkedIn after YouTube is maxed)

**What "new" does NOT mean:**
- Starting a new business before the current one is profitable
- Adding a new product line while the core product has issues
- Chasing a "new opportunity" because current work is hard

**New mistakes to avoid:**
- Launching new before core team is trained (core degrades while you're distracted)
- Adding complexity before systems exist to support it
- Funding "new" with cash that should be reinvested in "more"

## Applying More/Better/New to Revenue Decisions

### Revenue Model Decisions

| Situation | More/Better/New? | Action |
|---|---|---|
| Core offer is selling but not at full cadence | MORE | Increase outreach volume; post more content |
| Core offer is selling but close rate is 15% | BETTER | Improve sales script, proof assets, or offer positioning |
| Core offer is maxed; want recurring revenue | NEW | Add a continuity/subscription offer |
| Subscription has high churn | BETTER | Fix onboarding and delivery before adding new subscribers |
| Core and continuity are both performing | NEW | Add entry offer to capture lower-intent buyers |

### Pricing Decisions

| Situation | More/Better/New? | Action |
|---|---|---|
| Same price, same offer, want more money | MORE | Increase volume — more pitches, more ads |
| Volume is maxed, want more money per customer | BETTER | Raise prices, improve value stack |
| LTV is maxed at current offer tier | NEW | Add upsell or premium tier |

### Content Decisions

| Situation | More/Better/New? | Action |
|---|---|---|
| Content is converting but posting 2×/week | MORE | Post daily first |
| Posting daily but low engagement | BETTER | Improve hooks, better formats |
| Platform is maxed, want wider reach | NEW | Add second platform |

## The "New" Revenue Stream Sequence

When Phase 3 is confirmed, add revenue streams in this order:

```
1. CONTINUITY         → recurring version of the core offer (highest ROI addition)
2. UPSELL/PREMIUM     → higher-tier version for existing clients
3. ENTRY OFFER        → lower-ticket to capture unconverted leads
4. ADJACENT OFFER     → new problem, same ICP
5. NEW ICP            → same solution, new audience segment
6. NEW DELIVERY MODEL → same outcome, different format (DFY → DWY → DIY)
7. NEW BUSINESS MODEL → different revenue model entirely
```

**Never jump to step 7 before exhausting steps 1–6.**

## Common More/Better/New Violations (and their cost)

| Violation | What It Looks Like | The Cost |
|---|---|---|
| Adding "New" when "More" is the answer | Launching second product when first isn't at full volume | Diluted attention; neither succeeds |
| Optimizing "Better" when "More" is the answer | Rebuilding the website when outreach volume is 2/day | Premature perfection; no data to optimize |
| Refusing "New" when clearly ready | Grinding one channel past its ceiling | Capped growth; missed leverage |
| "New" without sufficient cash reserves | Funding new initiative from operating cash | Starves the core; both fail |
| "Better" before minimum viable proof | Optimizing conversion before 10 customers | Optimizing fiction |
